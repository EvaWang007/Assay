可以把三者统一为一个核心概念：

> **海流驱动的累计漂移建模（current-driven cumulative drift modeling）**

其中：

* 初始Bias是累计漂移的初始状态；
* 海流 (c_x,c_y) 是累计漂移的外部驱动力；
* 物理一致性约束规定漂移随时间如何演化。

它们不是三个孤立模块，而是同一个“漂移状态演化过程”的三个组成部分。

**一、统一理论表达**

设：

* (\mathbf p_t=[x_t,y_t]^\top)：AUV实际位置；
* (\mathbf p_t^{nom})：不考虑海流时的名义轨迹；
* (\mathbf d_t)：实际轨迹相对名义轨迹的累计漂移；
* (\mathbf c_t=[c_{x,t},c_{y,t}]^\top)：海流速度；
* (\mathbf b_0=[b_x,b_y]^\top)：预测起点的初始累计偏差。

定义：

[
\mathbf p_t=\mathbf p_t^{nom}+\mathbf d_t
]

其中累计漂移的初始状态为：

[
\mathbf d_0=\mathbf b_0
]

在最简单的二维运动学条件下，漂移状态满足：

[
\mathbf d_{t+1}
===============

\mathbf d_t+\Delta t,\mathbf c_t+\boldsymbol{\varepsilon}_t
]

其中 (\boldsymbol{\varepsilon}_t) 表示控制补偿误差、动力学耦合和未建模扰动。

展开后：

[
\mathbf d_{t+k}
===============

\underbrace{\mathbf b_0}*{\text{初始累计偏差}}
+
\underbrace{
\sum*{j=0}^{k-1}\Delta t
\begin{bmatrix}
c_{x,t+j}\
c_{y,t+j}
\end{bmatrix}
}*{\text{海流引起的时序累计漂移}}
+
\underbrace{
\sum*{j=0}^{k-1}\boldsymbol{\varepsilon}*{t+j}
}*{\text{未建模动态}}
]

这个公式就把Bias、(c_x/c_y) 和累计漂移统一起来了。

---

**二、物理一致性约束**

模型预测的漂移不能任意变化，而应满足上述状态转移规律。定义物理残差：

[
\mathbf r_t^{phy}
=================

## \hat{\mathbf d}_{t+1}

## \hat{\mathbf d}_t

\Delta t,\mathbf c_t
]

物理一致性损失为：

[
\mathcal L_{phy}
================

\frac{1}{T-1}
\sum_{t=1}^{T-1}
\left|
\hat{\mathbf d}_{t+1}
---------------------

## \hat{\mathbf d}_t

\Delta t,\mathbf c_t
\right|_2^2
]

最终训练目标：

[
\mathcal L
==========

\mathcal L_{traj}
+
\lambda_{phy}\mathcal L_{phy}
]

其中：

[
\mathcal L_{traj}
=================

\frac{1}{T}
\sum_{t=1}^{T}
\left|
\hat{\mathbf p}_t-\mathbf p_t
\right|_2^2
]

这样可以表述为：

> Bias和海流负责描述漂移从哪里开始、受到什么驱动，物理一致性损失负责约束该漂移如何随时间累积。

---

**三、和LSTM结合的完整表达**

LSTM负责从历史状态中学习名义运动规律和难以显式建模的非线性动态：

[
\mathbf h_t
===========

\operatorname{LSTM}
\left(
\mathbf s_{t-L+1:t}
\right)
]

将初始偏差和海流信息与LSTM特征融合：

[
\mathbf z_t
===========

\operatorname{Fusion}
\left(
\mathbf h_t,\mathbf b_0,\mathbf c_{t:t+T-1}
\right)
]

模型输出未来累计漂移或未来位置：

[
\hat{\mathbf d}_{t+1:t+T}
=========================

f_{\theta}(\mathbf z_t)
]

[
\hat{\mathbf p}_{t+k}
=====================

\hat{\mathbf p}*{t+k}^{nom}
+
\hat{\mathbf d}*{t+k}
]

因此整个模型的理论分工是：

| 部分           | 理论作用           |
| ------------ | -------------- |
| LSTM         | 学习AUV自身运动的时序规律 |
| 初始Bias       | 给出漂移状态的初始条件    |
| 海流 (c_x,c_y) | 给出漂移状态的外部驱动    |
| PI约束         | 约束漂移状态的时间演化    |
| PSO          | 优化模型超参数和损失权重   |

---

**四、论文中可以直接使用的理论叙述**

> 强海流环境下的AUV轨迹误差并非各预测时刻相互独立的随机误差，而是具有明显的时间累积特性。预测起始时刻已经存在的定位或跟踪偏差构成累计漂移的初始状态，而海流速度的水平分量 (c_x) 和 (c_y) 则作为外部扰动持续驱动漂移状态演化。因此，本文将初始偏差与海流影响统一建模为海流驱动的累计漂移状态。初始偏差决定漂移过程的初始条件，海流矢量决定后续漂移的方向和累积速率。

> 在此基础上，本文不将海流信息仅作为普通输入特征交由LSTM隐式学习，而是根据AUV运动学关系构造漂移状态转移方程，并将其转化为物理一致性约束。该约束限制相邻预测时刻之间的漂移增量应与同期海流作用保持一致，从而减少纯数据驱动模型产生的非物理轨迹变化。LSTM负责学习AUV自身运动及未建模非线性动态，累计漂移模型负责描述初始偏差和海流扰动的持续作用，两者共同形成海流感知物理一致性轨迹预测框架。

**五、建议的命名**

这一整体机制可以命名为：

> **海流驱动累计漂移物理一致性机制**

英文：

> **Current-Driven Cumulative Drift Physics-Consistency Mechanism**

模型则可以解释为：

> **BCPI-LSTM：Bias-compensated and Current-aware Physics-Informed LSTM**

论文中不要反复写“Bias模块 + Current模块 + PI模块”，而应统一写成：

> BCPI-LSTM通过构建海流驱动累计漂移状态，将初始偏差、海流扰动和物理一致性约束纳入同一状态演化框架。

需要注意：如果您的实际PI约束不是“相邻漂移增量等于海流位移”，而是完整的“AUV自身速度 + 海流速度”运动学约束，公式还应进一步写成：

[
\frac{\hat{\mathbf p}_{t+1}-\hat{\mathbf p}_t}{\Delta t}
========================================================

\mathbf R(\psi_t)\mathbf v_{b,t}
+
\mathbf c_t
]

最终论文必须采用与代码实现完全一致的物理方程。
