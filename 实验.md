我调查后的结论是：

> **目前没有发现一套同时满足“推进式AUV、高频轨迹、控制量、导航估计、真实位置、显式洋流 (u/v)”的完美公开数据集。最适合你的真实数据是 Gulf Stream 水下滑翔机数据；最适合你的模型训练方案是“DAVE仿真预训练 + Gulf Stream真实洋流验证/微调”。**

## 数据集排名

| 推荐 | 数据集                          |            明显洋流 | 轨迹 |              运动状态 | 适合用途          |
| -- | ---------------------------- | --------------: | -: | ----------------: | ------------- |
| 1  | Gulf Stream Spray Gliders    |    强，直接提供 (u/v) |  有 |                较少 | 真实洋流预测、真实场验证  |
| 2  | IOOS Glider DAC delayed-mode |    提供深度平均 (u/v) |  有 | pitch、roll、depth等 | 滑翔机等效漂移研究     |
| 3  | NPU AUV Navigation Dataset   |        没有显式洋流标签 |  有 |       INS、DVL、姿态等 | 推进式AUV轨迹/导航训练 |
| 4  | NOAA Hurricane Gliders       | 洋流环境明显，但字段依任务而异 |  有 |             CTD为主 | 补充真实海试任务      |
| 5  | DAVE Simulator               |     洋流完全可控且真值已知 | 完整 |                完整 | 生成你的高频训练集     |

## 1. 首选：Gulf Stream Spray Gliders

这是我认为最符合“有明显洋流”的真实数据。

滑翔机从Miami向Cape Cod航行，会多次横穿Gulf Stream并被洋流向下游平流。数据覆盖2004年至今，单次任务约100天、800个剖面、最大深度1000 m。([SprayData][1])

数据入口：[Gulf Stream Spray Glider Dataset](https://spraydata.ucsd.edu/projects/gs)

直接下载NetCDF：[GulfStream.nc](https://spraydata.ucsd.edu/erddap/files/binnedGS/GulfStream.nc)

ERDDAP在线筛选：[Gulf Stream ERDDAP](https://spraydata.ucsd.edu/erddap/tabledap/binnedGS.html)

### 关键字段

| 字段                        | 含义                 |
| ------------------------- | ------------------ |
| `mission`                 | 滑翔机任务编号            |
| `profile`                 | 剖面/潜航编号            |
| `time`                    | 剖面时间               |
| `latitude, longitude`     | 轨迹位置               |
| `depth`                   | 深度，10～1000 m       |
| `u, v`                    | ADCP测量的分层东向、北向水流速度 |
| `u_depth_mean`            | 每次潜航的深度平均东向流速      |
| `v_depth_mean`            | 每次潜航的深度平均北向流速      |
| `time_uv, lat_uv, lon_uv` | 深度平均流速对应的时间和位置     |

其分层流速范围约为：

[
u\in[-3.84,3.54]\ {\rm m/s},\qquad
v\in[-4.36,4.00]\ {\rm m/s}
]

深度平均流速最高超过 (2\ {\rm m/s})，洋流非常明显。数据包含约68个任务、40882个剖面。([Data Access Form][2])

### 最适合你的训练定义

输入过去 (T) 次潜航：

[
X_t=
[
lat,lon,time,depth,
u_{\rm mean},v_{\rm mean},
temperature,salinity
]_{t-T+1:t}
]

预测未来 (K) 次洋流：

[
Y_t=
[
u_{\rm mean},v_{\rm mean}
]_{t+1:t+K}
]

然后积分生成漂移：

[
\Delta p_{t+k}
==============

\sum_{i=1}^{k}
[u_i,v_i]\Delta t_i
]

限制是：它的时间尺度是“每次潜航/剖面”，通常为小时级，不适合直接训练你原来0.2秒级的TEB控制模型。

## 2. IOOS Glider DAC

IOOS提供大量公开水下滑翔机任务，可以通过API批量检索并用xarray读取NetCDF。官方示例展示了超过2400个deployment。([ioos.github.io][3])

数据入口：[IOOS Glider DAC](https://gliders.ioos.us/erddap/index.html)

一个可直接研究的OOI任务：

[OOI Gulf of Alaska Glider](https://gliders.ioos.us/erddap/tabledap/gp_363-20160630T0230-delayed.html)

字段包括：

```text
precise_time
precise_lat / precise_lon
depth
pitch / roll
u / v
time_uv / lat_uv / lon_uv
```

其中 `u/v` 明确定义为滑翔机水下航段的深度平均流速。该示例流速约为：

[
u\in[-0.104,0.115],\qquad
v\in[-0.180,0.083]\ {\rm m/s}
]

([Data Access Form][4])

它比Gulf Stream流速弱，但字段更接近你的历史状态输入。

## 3. 推进式AUV：NPU真实导航数据集

GitHub：[AUV Navigation Dataset](https://github.com/nature1949/AUV_navigation_dataset)

论文：[Underwater AUV Navigation Dataset in Natural Scenarios](https://www.mdpi.com/2079-9292/12/18/3788)

特点：

* 真实湖泊和海洋实验；
* 总航程约147 km；
* 包含光纤惯导、DVL、深度和姿态等导航信息；
* CSV格式；
* CC0许可证，可以自由用于训练。([GitHub][5])

它的优势是高频推进式AUV运动状态更贴合你的模型；缺点是没有明确提供洋流 (u/v) 真值。

如果数据里同时存在航位推算位置和上浮GPS，可以按潜航段构造：

[
c^{eq}
======

\frac{
p_{\rm GPS,end}
---------------

p_{\rm nav,end}
}{\Delta T}
]

但这只能称为“等效漂移速度”，包含洋流、导航误差和模型误差。

## 4. NOAA Hurricane Gliders

NOAA公开了加勒比海、北大西洋和墨西哥湾的大量滑翔机任务，每次潜航数据以NetCDF提供。([CARICOOS Hurricane Underwater Gliders][6])

入口：[NOAA Hurricane Glider Data](https://www.aoml.noaa.gov/phod/goos/gliders/data.php)

这些任务所处环境海流和风暴影响明显，但很多文件主要包含温度、盐度、深度和位置，不保证每次任务都有可用的流速标签，因此优先级低于Gulf Stream数据。

## 5. 最适合生成训练集：DAVE

DAVE是开源水下机器人仿真环境，支持：

* 恒定洋流；
* 分层洋流；
* 潮汐振荡；
* 时空变化洋流；
* AUV动力学；
* DVL、IMU和导航传感器；
* 完整真实状态与控制输入。([arxiv.org][7])

项目：[DAVE Simulator](https://field-robotics-lab.github.io/dave.doc/)
洋流教程：[DAVE Ocean Current Plugin](https://field-robotics-lab.github.io/dave.doc/contents/dave_env/Ocean-Current/)

它最适合生成：

```text
time
true_x / true_y / true_z
nav_x / nav_y / nav_z
surge / sway / heave
roll / pitch / yaw
thruster commands
current_u / current_v / current_w
```

## 最终建议

你的论文最好采用三层数据：

1. **DAVE仿真训练集**
   生成0.2秒或1秒级AUV轨迹，拥有精确洋流和等效扰动真值。

2. **Gulf Stream真实洋流数据**
   用真实 `u/v` 构造时空变化流场，替换简单的正弦或恒定洋流。

3. **NPU真实AUV数据**
   验证模型对真实推进式AUV传感器噪声和运动状态的适应能力。

最合理的组合是：

[
\boxed{
\text{Gulf Stream真实流场}
+
\text{DAVE AUV动力学}
\rightarrow
\text{高频、有洋流真值的训练轨迹}
}
]

然后再用真实滑翔机数据做外部验证。这样比只找一个现成CSV更符合你“LSTM等效扰动预测 + 物理积分 + TEB补偿”的研究目标。
